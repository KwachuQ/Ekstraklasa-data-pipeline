1. Cel główny: stworzyć bazę danych wiarygodnych statystyk dotyczących meczów rozegranych w Ekstraklasie, żeby móc przygotowywać raporty przedmeczowe
i na ich podstawie szacować prawdopodobieństwo końcowego wyniku przyszłych meczów

2. Cele pomocnicze:
- stworzenie data pipeline z API Sofascore, który automatycznie aktualizuje bazę danych po każdej kolejce
- upewnienie się co do poprawności, kompletności i spójności statystyk podstawowych (tych pobranych z API)
- stworzenie na ich podstawie bardziej zaawansowanych agregatów statystycznych (forma drużyny, statystyka np. XG per 90 minut etc.)
- stworzenie modelu ML do przewidywania kursu na końcowy wynik (prawdopodobieństwo przeliczone na kurs dziesiętny tzw. fair odds)
- przygotowanie dashboardów w metabase do prezentacji tych statystyk w interaktywnej formie (wybór drużyny, zakresu danych, zakresu sezonów etc.)

3. Data pipeline

- pierwsza faza: import wszystkich danych historycznych i ich transformacja
- druga faza: orkiestracja automatycznego, regularnego pobierania danych z ostatniej kolejki

dodatkowo: stworzenie kompletnego modułu do obsługi API Sofascore w pythonie

4. Walidacja danych

- testy jakości danych w warstwie bronze i silver (null values, duplikaty, outliers)
- czyszczenie danych z meczów przełożonych i odwołanych
